# 気象庁情報取得・JSON変換プログラム 運用ガイド

本プログラムは、気象庁の XML フィードを取得し、JSON 形式に解析して提供するものです。

## 1. セットアップ

### 必要条件
- Node.js (v14以上推奨)
- npm

### インストール
```bash
npm install
```

## 2. テスト・動作確認

### サンプルデータによるパース検証
提供されている過去の地震・津波 XML サンプルを使用して、正しく JSON に変換されるかを確認できます。
```bash
node test_parsers.js
```
出力に `✅ パース成功` と表示されれば、ロジックは正常です。

### サーバー立ち上げテスト
ローカル環境で API サーバーを起動します。
```bash
node jma_server.js
```
ブラウザなどで `http://localhost:443/jma/latest` にアクセスし、現在の最新情報が含まれる配列が返ってくるか確認してください。

---

## 3. 本番環境での使用方法

### プロセス管理 (PM2 の推奨)
本番環境では、サーバーの停止を自動復旧するため、`PM2` などのプロセス管理ツールの使用を推奨します。

```bash
# PM2のインストール
npm install pm2 -g

# サーバーの起動
pm2 start jma_server.js --name jma-api

# 自動起動設定（OS再起動時用）
pm2 startup
pm2 save
```

### 運用上の注意点

#### ① 訓練報・試験報の除外
気象庁は定期的に訓練や試験の電文（`status` が `訓練` や `試験`）を送信します。
- 本番のシステムで使用する際は、必ず JSON 内の `status === "通常"` であることを確認してください。

#### ② キャッシュと保持期間
- `jma_server.js` には 1 分間の更新キャッシュ機能が実装されています。
- **10分間保持**: 取得した震災情報は、サーバーが取得してから**10分間**キャッシュに保持され、APIの配列に含まれ続けます。
- **古い情報の収集防止**: サーバー起動時にフィード内に10分以上前の古い情報がある場合、それらは無視されます。

#### ③ 監視とログ
- `pm2 logs jma-api` でエラーメッセージを確認できます。
- ネットワークエラー（気象庁サーバーへの接続失敗）に備え、プログラム内でもリトライ処理や古いキャッシュの保持を考慮しています。

## 4. データ構造の概要

| フィールド | 内容 |
| :--- | :--- |
| `status` | 運用ステータス（通常 / 試験 / 訓練） |
| `infoType` | 情報の種類（発表 / 訂正 / 取消） |
| `earthquake` | 震源情報（発生時刻、震央地名、マグニチュード等） |
| `intensity` | 震度情報（最大震度、各地の震度、長周期地震動階級） |
| `tsunami` | 津波情報（警報状況、到達予想、満潮時刻、観測値） |
