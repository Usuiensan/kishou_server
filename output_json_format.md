# JMA API 出力JSON形式の仕様

本ドキュメントでは、JMA API (`/jma/latest`) が出力するJSON形式について解説します。
この形式は、特定の表示端末やVRChatなどの外部アプリケーションでの利用を想定しています。

## 概要
APIは常に一つのオブジェクトを返却します。情報がある場合は最新のものを、ない場合は「安定」状態を示すデフォルト値を返します。

## 共通フィールド
| フィールド | 型 | 説明 |
| :--- | :--- | :--- |
| `type` | `String` | 内容の種類（表示の優先度や色分けに使用） |
| `id` | `String/Number` | 情報の一意なID (EventIDなど) |
| `lines` | `Array<String>` | 表示用のテキスト行（改行を含む場合あり、一部リッチテキストタグを含む） |
| `timestamp` | `String` | 情報が処理された日時 (ISO 8601形式) |

---

## 種類別詳細 (`type`)

### 1. 地震情報 (`type`: `earthquake_*` / `emergency`)
震度に応じた種別が設定されます。
- `earthquake_1` : 最大震度 1
- `earthquake_2` : 最大震度 2
- `earthquake_3` : 最大震度 3
- `earthquake_4` : 最大震度 4
- `emergency` : 最大震度 5弱 以上、震度速報、または緊急地震速報
- `earthquake` : 震源情報のみ（震度情報がない場合）

### 2. 津波情報 (`type`: `emergency`)
津波警報・注意報が発表されている場合に返却されます。
- `type`: `emergency` 固定
- 警報の種類（大津波警報・津波警報・津波注意報）が `lines` 内でカラーコード `<color=#...>` と共に出力されます。

### 3. 気象情報 (`type`: `weather`)
特別警報、警報、線状降水帯などの情報がある場合に返却されます。
- `type`: `weather` 固定
- `lines` 内で見出しや種類（[警報]など）がカラーコード `<color=#...>` と共に出力されます。

### 4. 安定状態 (`type`: `stable`)
現在、有効な地震・津波情報がない場合のデフォルト値です。
- `type`: `stable`
- `id`: `"none"`
- `lines`: `["現在、発表されている地震・津波情報はありません。"]`

---

## `lines` フィールドの特記事項

### カラーコード
一部の行では、以下のようなリッチテキスト形式のカラーコードが含まれる場合があります。
- 例: `<color=#FF00FF>[大特別警報] ...</color>`
- 例: `<color=#FF2800>[津波警報]</color>`

### エリアの折り返し
一つの震度や警報に複数の地域が含まれる場合、可読性のために一定数（デフォルトは1行6件程度）で分割して複数の要素として格納されます。
